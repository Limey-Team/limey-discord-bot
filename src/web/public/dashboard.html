Welcome to the Limey Dashboard

          
Your bot management command center


          

            
This dashboard gives you full control over your Limey Discord bot. Monitor events in real time, search logs, configure settings, manage tickets, and run custom bot instances — all from one place.

          


          

            

              ⌨️
              

                Keyboard Shortcut
                
Press ? anywhere in the dashboard to open this guide.

              

            

            

              🎫
              

                Ticket System
                
Manage tickets and configure panels, options, and questions visually.

              

            

            

              🤖
              

                Custom Bots
                
Add your own bot tokens to run custom bots alongside Limey.

              

            

          

        


        
        

          
◎ Live Feed

          
Real-time event monitoring


          

            
The Live Feed displays bot events as they happen in real time via Server-Sent Events (SSE). Every action the bot detects — messages, bans, joins, voice activity — appears here instantly.

          


          
What you can do

          
    
            
  • Filter by event type or search within event data
    • 
            
  • Pause the feed to freeze the current view without missing new events
    • 
            
  • Click any entry to inspect its full JSON details
    • 
            
  • Limit the server view using the guild selector if you manage multiple servers
    • 
          


          

            💡 Tip: The feed auto-scrolls as new events arrive. Use the pause toggle to examine a specific moment without interruption.
          

        


        
        

          
≡ Explore Logs

          
Search and analyze historical events


          

            
The Explore Logs tab lets you search through all recorded events with powerful filters. Paginated results make it easy to browse large volumes of data.

          


          
How to use

          
    
            
  • Pick an event type from the dropdown to narrow results
    • 
            
  • Search by guild name, channel, or user in the text fields
    • 
            
  • Use the Search data field to find specific content within event payloads
    • 
            
  • Click any row to open the full JSON detail view
    • 
            
  • Navigate pages with the ← Prev / Next → controls
    • 
          


          

            💡 Tip: Press Enter in any filter field to trigger the search immediately.
          

        


        
        

          
◈ Statistics

          
System metrics and event distribution


          

            
The Stats tab gives you a bird's-eye view of your bot's activity: total events logged, oldest/newest entries, rate limit hits, and a visual breakdown of events by type.

          


          
Dashboard metrics

          
    
            
  • Total Logs — count of every event ever recorded
    • 
            
  • Event Types — how many distinct event categories exist
    • 
            
  • Rate Limits — number of Discord API rate limits hit, with per-route breakdown
    • 
            
  • Bar Chart — visual comparison of the top 20 event types by volume
    • 
          

        


        
        

          
🔔 Log Event Configuration

          
Control which events are sent to your Discord log channel


          

            
The Log Config tab lets you fine-tune which Discord events get forwarded to your server's log channel. Each event type can be individually enabled or disabled.

          


          
How to use

          
    
            
  • Select a server from the dropdown to see its current event configuration
    • 
            
  • Toggle individual events using the switches next to each event name
    • 
            
  • Use Enable all / Disable all buttons on each group to toggle entire categories
    • 
            
  • Use the top Enable All / Disable All buttons to control all events at once
    • 
            
  • The badge in the header shows how many events are currently enabled
    • 
          


          

            💡 Tip: Disabling an event in the log config only stops it from being sent to your Discord log channel. All events are still recorded and visible in the Live Feed and Explore tabs.
          

        


        
        

          
⚙ Settings

          
Log channel configuration


          

            
The Settings tab lets you configure which Discord channel Limey sends event log messages to, giving your moderation team a live feed of bot activity right inside your server.

          


          
How it works

          
    
            
  1. Select a server from the dropdown
    2. 
            
  2. Choose a text channel for log messages
    3. 
            
  3. Click Save to apply — logs will start flowing immediately
    4. 
            
  4. Use Disable Logging to turn it off for that server
    5. 
          

        


        
        

          
🎫 Tickets

          
Monitor and manage support tickets


          

            
The Tickets tab displays all open and closed support tickets across your servers. See who created each ticket, who claimed it, its priority, and relevant timestamps.

          


          
Features

          
    
            
  • Live stats bar — totals, open, closed, claimed, and pinned counts
    • 
            
  • Filter by status (All / Open / Closed / Claimed / Pinned)
    • 
            
  • Search by creator ID, channel ID, or option type
    • 
            
  • Color-coded cards — green border for open, red for closed, blue for claimed
    • 
          


          

            💡 Tip: Use the status filter dropdown to quickly find unclaimed open tickets that need attention.
          

        


        
        

          
🔧 Ticket Configuration

          
Configure the entire ticket system


          

            
This is the control center for the ticket system. Everything from panel definitions to modal questions to priority levels can be edited here. All changes are saved to files and auto-committed to GitHub.

          


          
Sub-tab guide

          
    
            
  • General — Core settings: permissions, category IDs, cooldowns, limits
    • 
            
  • Panels — Define the embed messages and buttons users click to create tickets
    • 
            
  • Options — Configure ticket types (support, report, application, etc.)
    • 
            
  • Questions — Modal questions shown before a ticket is created
    • 
            
  • Priorities — View available priority levels (Low / Medium / High / Urgent)
    • 
            
  • Transcripts — Browse saved ticket transcripts from closed/deleted tickets
    • 
            
  • Spawn Panel — Send a panel to any text channel without typing commands
    • 
          


          

            💡 Tip: Use the Spawn Panel sub-tab to send a ticket panel to any channel — no need to run /ticket-panel in Discord!
          

        


        
        

          
📬 Modmail

          
Private messaging between users and staff


          

            
The Modmail system allows users to privately message server staff by DMing the bot. Each conversation gets its own private channel in a designated category where staff can respond. Replies are forwarded back to the user via DM — creating a seamless two-way private conversation.

          


          
How it works

          
    
            
  1. Admin runs /modmail setup enable — selects a category, staff role, and optional log channel
    2. 
            
  2. Users DM the bot — a private channel is created in the modmail category
    3. 
            
  3. Staff see the message and reply directly in the channel
    4. 
            
  4. The bot forwards replies back to the user anonymously or with staff name
    5. 
            
  5. Staff can close the thread with /modmail close or click the Close button
    6. 
          


          
Key Features

          
    
            
  • Anonymous replies — toggle per-thread so staff names stay hidden from users
    • 
            
  • Auto-close — threads close automatically after configurable hours of inactivity
    • 
            
  • Cooldown — prevent users from spamming new threads after one is closed
    • 
            
  • Block/Unblock — block specific users from using modmail entirely
    • 
            
  • Transcripts — generate a .txt transcript of any thread with /modmail transcript
    • 
            
  • Staff alerts — pings a role and sends notifications to a log channel when new modmail arrives
    • 
            
  • Custom greeting — customize the embed title, description, and color shown in new threads
    • 
            
  • Auto-reply — send an automatic acknowledgment when users first DM the bot
    • 
          


          
Available Commands

          
    
            
  • Setup: /modmail setup enable, /modmail setup disable, /modmail setup category, /modmail setup staffrole, /modmail setup logchannel, /modmail setup alertrole, /modmail setup cooldown, /modmail setup autoclose, /modmail setup greeting, /modmail setup autoreply, /modmail setup view
    • 
            
  • Actions: /modmail close, /modmail reopen, /modmail anon, /modmail block, /modmail unblock, /modmail blocked, /modmail transcript, /modmail stats
    • 
          


          

            💡 Tip: Use the Close, Anonymous, and Block User buttons at the top of every modmail staff channel for quick actions without typing commands.
          

        


        
        

          
🤖 Custom Bots

          
Run your own bot instances alongside Limey


          

            
The Custom Bots tab allows server owners to add their own Discord bot tokens. Each bot runs in the same process, registers its own slash commands, and operates independently for its assigned server.

          


          
How to add a bot

          
    
            
  1. Go to the Discord Developer Portal and create or select an application
    2. 
            
  2. Go to the Bot page and click Reset Token to reveal your token
    3. 
            
  3. Select your server from the dropdown
    4. 
            
  4. Paste the token and click Start Bot
    5. 
          


          
Important notes

          
    
            
  • Tokens are never stored on disk — they kept in memory only
    • 
            
  • On server restart, bots started from the dashboard won't auto-restart
    • 

  • For restart persistence, set BOT_TOKEN_GUILDID as an environment variable
          
            

          

        


        
        

          
💾 Backups

          
Full data snapshot & restore system


          

            
The Backups system creates complete snapshots of all bot data — config, warnings, ticket configurations, ticket data, transcripts, and custom bot metadata. Each backup is stored as a full copy so you can restore to any point in time.

          


          
What gets backed up

          
    
            
  • config.json — All guild settings (log channels, verify roles, trap configs)
    • 
            
  • warnings.json — All warning records for every user
    • 
            
  • Ticket configs — General, panels, options, questions, transcripts settings
    • 
            
  • Ticket data — All open/closed tickets, transcripts, and stats
    • 
            
  • Bot metadata — Custom bot registrations (tokens are not stored)
    • 
          


          
How to use

          
    
            
  • Create a backup via /backup [label] or click "Create Backup" in the dashboard
    • 
            
  • View backups with /backups or in the Backups dashboard tab
    • 
            
  • Restore with /restore id:<id> confirm:true — requires double confirmation
    • 
            
  • Delete old backups from the dashboard to save space
    • 
          


          

            💡 Tip: Create a backup before making major configuration changes (e.g., before reconfiguring tickets or clearing warnings). This way you can always roll back if something goes wrong.
